How does JS work

javascript is a single thread language which can do one task at a time. In order to do that, it puts all functions in a file in a list called the call stack and executes the functions one after the other. Synchronous functions follow the order of the code. But asynchronous functions wait for all synchronous functions to execute and then executes.

JS event loop

Event loop is the way JS manages and runs the code. The HEAP stores all of the different objects in the memory. The call stack executes the code and the Queue keeps the parameters of the functions. When a messege form the queue is handeled by the event loop, it is removed from the queue and the corresponding fucntion takes the messege as a parameter and runs.

Difference between local and session storage.

The diffrerence between local and session storage is local storage stays in the memory even after restarting the browser or the tab. But sesssion storage expires after a session is over. A session is over when the browser tab is closed or the browser itself is closed. After we close the tab or the browser and reopen the page, the session storage from the previous session will not be there but the local storage will.
